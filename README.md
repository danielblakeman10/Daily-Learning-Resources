## 🎯 What Is This?

This is a **structured, self-guided learning path** for mastering **AWS DevOps, Platform Engineering, and Cloud Infrastructure**. Whether you're preparing for the AWS DevOps Engineer Professional exam, building a career in cloud infrastructure, or upskilling your team — this repo gives you a clear roadmap with actionable exercises, real-world workflows, and curated free resources.

### ✨ Why This Repo?

| Feature | Description |
|---|---|
| **🗺️ 12-Pillar Curriculum** | Covers the complete AWS DevOps landscape from foundations to advanced topics |
| **⚙️ Real Workflows** | Maps each topic to actual DevOps practices you'll use on the job |
| **🎓 Free Resources Only** | AWS Docs, Well-Architected, Hands-On Tutorials — all free |
| **📊 Progress Tracking** | Built-in checklist so you can track your learning journey |
| **🏋️ Hands-On Exercises** | Practical exercises to cement your knowledge |

## 🏛️ The 12-Pillar Curriculum

```
AWS DevOps & Platform Engineering
│
├── 🏗️  Pillar 01 — Foundation & Governance
│       Goal: IaC, organizational units, SCPs, service catalog
│       Services: CloudFormation, AWS Config, Control Tower, Organizations, Service Catalog
│
├── 🔐  Pillar 02 — Identity, Security & Compliance
│       Goal: IAM, SSO, guardrails, audit trails, compliance frameworks
│       Services: IAM, SSO, GuardDuty, Security Hub, Audit Manager, KMS
│
├── 🌐  Pillar 03 — Networking & Hybrid Architecture
│       Goal: VPC design, connectivity, DNS, global routing
│       Services: VPC, Transit Gateway, Direct Connect, Route 53, VPN, Cloud WAN
│
├── 💻  Pillar 04 — Compute & Application Hosting
│       Goal: EC2, auto-scaling, load balancing, container compute
│       Services: EC2, Auto Scaling, ELB, Fargate, App Runner, Outposts
│
├── 💾  Pillar 05 — Storage & Data Strategy
│       Goal: S3 tiering, EBS/EFS, data lifecycle, backup strategy
│       Services: S3, EBS, EFS, FSx, Storage Gateway, Data Lifecycle Manager
│
├── 🗄️  Pillar 06 — Databases & Data Layer
│       Goal: RDS, Aurora, DynamoDB, ElastiCache, data migration
│       Services: RDS, Aurora, DynamoDB, ElastiCache, DocumentDB, Neptune, DMS
│
├── ⚡  Pillar 07 — Serverless & Event-Driven
│       Goal: Lambda architecture, event buses, message queuing
│       Services: Lambda, EventBridge, SQS, SNS, Step Functions, Kinesis
│
├── 🔧  Pillar 08 — DevOps & Automation
│       Goal: CI/CD pipelines, GitOps, release strategies, change management
│       Services: CodePipeline, CodeBuild, CodeDeploy, CodeArtifact, Systems Manager, CloudCenter
│
├── 🐳  Pillar 09 — Containers & Kubernetes
│       Goal: EKS, ECS, Fargate, Helm, container security
│       Services: EKS, ECS, ECR, Fargate, Elastic Load Balancer, App Mesh
│
├── 📊  Pillar 10 — Observability & Monitoring
│       Goal: Centralized logging, metrics, tracing, alerting
│       Services: CloudWatch, X-Ray, CloudTrail, OpenSearch, DevOps Guru, EventBridge
│
├── 🛡️  Pillar 11 — Security & DevSecOps
│       Goal: Shift-left security, compliance-as-code, secrets management
│       Services: Secrets Manager, Parameter Store, Inspector, Artifact, Inspector, Macie
│
└── 🚀  Pillar 12 — Migration & Cost Optimization
        Goal: Database migration, app lift-and-shift, FinOps practices
        Services: DMS, MGN, Application Migration Service, Cost Explorer, Trusted Advisor, RAIs
```

## 🚀 How to Use This Repo

### For Self-Guided Learners

1. **Start at Pillar 01** — The curriculum is sequential; each pillar builds on the previous
2. **Read the pillar doc** — Covers learning objectives, key services, and resources
3. **Follow the recommended resources** — All free: AWS Docs, Well-Architected, Hands-On tutorials
4. **Complete the practical exercises** — Hands-on labs to cement your knowledge
5. **Check off progress** — Update the checkbox in the table above
6. **Repeat** — Aim for 30-60 minutes per session

### For Teams & Managers

1. Assign specific pillars to team members
2. Use the practical exercises as team workshop topics
3. Track progress using the progress tracker
4. Review pillar docs in 1:1s to discuss learnings

### Suggested Learning Path

```
Beginner → Intermediate → Advanced

01 Foundation → 03 Networking → 04 Compute → 06 Databases
                                                  ↘ 08 DevOps
02 Identity →  07 Serverless → 09 Containers → 10 Observability
                                               ↘ 11 Security
05 Storage →                              12 Migration & Cost
```
